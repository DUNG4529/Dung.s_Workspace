

### Giai đoạn 1: Phân tích bài toán (Decomposition & Pattern Recognition)

Trước khi bắt tay vào thiết kế, bạn cần bóc tách các yêu cầu của bài toán thành các danh từ (đối tượng) và động từ (hành động):

- **Danh từ (Đối tượng dữ liệu):** Employee (Nhân viên), Role (Developer, Tester, Manager, HR), File `employees.txt`.
- **Các thuộc tính và ràng buộc (Constraints):**
    - _ID:_ Không rỗng, duy nhất, định dạng E + 3 chữ số (VD: E001).
    - _Name:_ Không rỗng.
    - _Role:_ Chỉ được là Developer, Tester, Manager, HR.
    - _Base Salary:_ Số dương.
    - _Working Days:_ Số nguyên từ 0 - 26.
    - _Bonus:_ Lớn hơn hoặc bằng 0.
    - _Status:_ active hoặc inactive.
- **Động từ (Hành vi/Chức năng):** Load từ file, Add, Update, Remove, Search, Calculate payroll, Display, Save xuống file.

### Giai đoạn 2: Thiết kế kiến trúc & Vẽ UML (Abstraction)

Thay vì code toàn bộ vào một file main, bạn cần áp dụng mô hình thiết kế như **n-Layers** hoặc **MVC** để đạt tiêu chí "Low Coupling - High Cohesion".

- **Models Layer / Model:** Lớp `Employee` chứa các thuộc tính trên, tạo Constructors, Getters/Setters, và phương thức `toString()`.
- **Neutral (Tools) Layer:**
    - Lớp `Inputter`: Chuyên xử lý việc nhập dữ liệu từ bàn phím (nhập chuỗi, số nguyên, số thực) và yêu cầu nhập lại nếu sai.
    - Lớp interface `Acceptable`: Định nghĩa các hằng số Regular Expression (Regex) để Validate dữ liệu (VD định dạng ID `^E\d{3}$`, Role, Status...) và phương thức `isValid()`.
- **Business Layer / Controller:** Lớp `EmployeeManager` (sử dụng Collection như `List<Employee>` hoặc `Map<String, Employee>`) chứa các logic chính: thêm, sửa, xóa, tính lương, và đọc/ghi file. Bạn cần chú ý xử lý ngoại lệ (Exception) khi đọc file text vì có thể gặp dòng dữ liệu bị lỗi.
- **Vẽ UML:** Từ kiến trúc trên, bạn vẽ Class Diagram thể hiện rõ các lớp, các thuộc tính (kèm modifier `private`/`public`), và mối quan hệ giữa chúng.

### Giai đoạn 3: Thiết kế Thuật toán & Lưu đồ (Algorithm Flowcharts)

- Liệt kê tuần tự các bước xử lý cho những hàm phức tạp.
- Bạn cần vẽ Lưu đồ (Flow-chart) cho các chức năng cốt lõi. Ví dụ: Lưu đồ cho chức năng `Add new employee` (Nhập -> Validate bằng Regex -> Check trùng ID -> Khởi tạo đối tượng -> Đưa vào List) hoặc chức năng `Calculate monthly payroll`.

### Giai đoạn 4: Triển khai Code (Coding)

Code theo thứ tự từ các lớp nền tảng lên các lớp xử lý:

1. Code interface `Acceptable` (viết các Regex kiểm tra dữ liệu).
2. Code lớp `Inputter` (Hàm nhập số, chuỗi, validate với Regex).
3. Code lớp `Employee` (Model).
4. Code lớp `EmployeeManager` (Các chức năng CRUD, tính lương, đọc/ghi file `employees.txt`).
5. Code lớp `Main` (View/Menu) để người dùng tương tác.
6. _Lưu ý code style:_ Đặt tên biến theo chuẩn camelCase, dùng danh từ cho biến/lớp, động từ cho hàm, và viết comment rõ ràng.

### Giai đoạn 5: Viết AI Audit Log (Song song với lúc làm project)

Với môn LAB211 (dạng Assignment), bạn cần lưu ý các yêu cầu **bắt buộc** đối với AI Audit Log:

- **Số lượng:** Cần có từ **16 - 24 Core Prompts**.
- **Phát hiện ảo giác (Hallucination):** Phải chủ động phát hiện ít nhất **2 trường hợp** AI nói sai/bịa đặt (ví dụ: gợi ý code dùng thư viện không tồn tại, hoặc logic sai sót không xét đến các ràng buộc của bài toán).
- **Cách viết Human Delta:** Mỗi entry quan trọng phải trả lời đủ 4 câu hỏi: (1) Trọng tâm tư duy phản biện (AI đúng/sai chỗ nào?), (2) Bối cảnh thực tế AI không biết, (3) Bạn đã thay đổi/tối ưu ra sao, (4) Quyết định cuối cùng của bạn. Bạn sẽ dùng Template Excel quy định của trường để điền.
